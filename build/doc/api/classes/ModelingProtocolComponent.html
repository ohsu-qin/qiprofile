public getLabel(property): String

Formats the protocol property labels. The top-level properties, .e.g. configuration, display using the default getLabel transformer. The configuration properties are formatted as the underscored unqualified input property name, e.g. r10_val for input r1.r10Val, as opposed to the default getLabel transformer.

Parameters:

Name Type Attribute Description
property String

the input property path

Return:

String

the unmodified property name

Example:

A protocol with nested configuration propertyr1.r10_val displays as:

  +---------+----------------+
  |  Configuration           |
  +---------+----------------+
  |   r1                     |
  | +---------+------------+ |
  | | r10_val +   ....     | +
  | +---------+------------+ |
  | + ....                 | |
  | +---------+------------+ |
  +---------+----------------+

The input property name is the camelCase transformation of the REST input JSON object property, which is customarily obtained from a Python imaging pipeline. In that case, this getLabel function restores the upstream Python property naming convention.
